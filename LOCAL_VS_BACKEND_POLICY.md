# ✅ تصحيح سياسة الحفظ - Local vs Backend\n\n## 📋 السياسة الجديدة:\n\n### ✅ **حفظ محلي فقط** (localStorage):\n```\n✓ avatarDataUrl         - الصورة قبل الرفع\n✓ emergencyContacts     - جهات الاتصال الطارئة\n✓ sosMessage            - رسالة SOS\n```\n\n### ✅ **حفظ في Backend فقط** (API):\n```\n✓ fullName              - الاسم الكامل\n✓ nationalId            - رقم الهوية\n✓ phoneNumber           - رقم الهاتف\n✓ region                - المنطقة/المحافظة\n✓ avatarUrl             - الصورة المرفوعة (من API)\n✓ assistancePreferences - تفضيلات المساعدات\n✓ assistanceLocation    - موقع المساعدة\n✓ assistanceRadius      - نطاق المساعدة\n```\n\n---\n\n## 🔄 تدفق البيانات:\n\n### عند تعديل **الاسم** (بيانات API):\n```\nUser edits name\n    ↓\nClick Save\n    ↓\n❌ NOT saved locally\n    ↓\n✅ ONLY sent to Backend\n    ↓\nBackend validates & saves\n    ↓\nRefresh data from API\n    ↓\nUI updates with API data ✅\n```\n\n### عند رفع **الصورة** (بيانات محلية):\n```\nUser selects image\n    ↓\nCrop image locally\n    ↓\n✅ Save avatarDataUrl to localStorage\n    ↓\nShow preview\n    ↓\nClick Save\n    ↓\n✅ Send to Backend for upload\n    ↓\nBackend uploads & returns avatarUrl\n    ↓\n✅ Cache still has local copy (as fallback)\n    ↓\nUI shows new avatar from Backend ✅\n```\n\n---\n\n## 🛠️ الملفات المعدلة:\n\n### 1. `lib/profile/localProfileStorage.ts`\n```\n❌ قبل: حفظ fullName, nationalId, phoneNumber, region, etc.\n✅ بعد: حفظ فقط avatarDataUrl, emergencyContacts, sosMessage\n```\n\n### 2. `hooks/useProfile.ts`\n```\n❌ قبل: saveLocalProfileData(id, { fullName, nationalId, ... })\n✅ بعد: saveLocalProfileData(id, { avatarDataUrl })\n      ← فقط الصورة المحلية\n```\n\n### 3. `lib/profile/localProfileStorage.ts` - mergeProfileAvatarOnly()\n```\n❌ قبل:\nreturn {\n  ...profile,\n  fullName: profile.fullName || local.fullName,     // ❌\n  nationalId: profile.nationalId ?? local.nationalId,  // ❌\n  ...\n}\n\n✅ بعد:\nreturn {\n  ...profile,\n  avatarUrl: local.avatarDataUrl ?? profile.avatarUrl,  // ✅\n  // ✅ جميع البيانات الأخرى من API فقط\n}\n```\n\n---\n\n## 💡 الفوائد:\n\n### 1. **مصدر واحد للحقيقة**\n```\n✅ البيانات المهمة من Backend فقط\n✅ لا توجد نسخ متعددة من نفس البيانات\n✅ لا توجد تضارب بين localStorage والـ API\n```\n\n### 2. **عزل البيانات المحلية**\n```\n✅ الصور المحلية كـ cache فقط (قبل الرفع)\n✅ جهات الاتصال محفوظة محلياً (لا ترسل للـ API)\n✅ SOS message محفوظة محلياً (خصوصية)\n```\n\n### 3. **أمان أفضل**\n```\n✅ البيانات الحساسة من Backend مباشرة\n✅ لا نعتمد على localStorage للبيانات المهمة\n✅ في حالة مسح localStorage، البيانات محفوظة في Backend\n```\n\n### 4. **تجربة مستخدم أفضل**\n```\n✅ صور محلية تظهر مباشرة (UX سريعة)\n✅ التحديثات من Backend تظهر عندما تصل (data current)\n✅ لا توجد نسخة قديمة تظهر من localStorage\n```\n\n---\n\n## 🧪 أمثلة الاختبار:\n\n### Test 1: تعديل الاسم\n```\n1. غيّر الاسم من \"أحمد\" إلى \"أحمد محمد\"\n2. اضغط حفظ\n3. افتح DevTools → Application → LocalStorage\n   → لا يوجد \"fullName\" ❌\n4. أعد تحميل الصفحة\n   → الاسم الجديد يظهر من API ✅\n5. مسح localStorage بالكامل\n   → الاسم محفوظ في الخادم ✅\n```\n\n### Test 2: رفع صورة\n```\n1. اختر صورة جديدة\n2. أعد التصغير\n3. قبل الحفظ:\n   → الصورة تظهر كـ preview ✅\n   → avatarDataUrl محفوظة محلياً ✅\n4. اضغط حفظ\n5. الصورة ترسل للـ Backend\n6.
