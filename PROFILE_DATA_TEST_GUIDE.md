# 📋 اختبار حفظ الصور والبيانات الشخصية

## 🚀 الاختبار 1: حفظ الصورة والعودة

### الخطوات:
```
1. افتح التطبيق
2. سجل دخول بحساب عادي
3. اذهب إلى صفحة تعديل الملف الشخصي (/profile/edit)
4. اضغط على زر \"تعديل\" على الصورة الشخصية
5. اختر صورة من جهازك
6. اقطع الصورة حسب الرغبة
7. اضغط \"تأكيد\"
8. اضغط \"حفظ التعديلات\"
9. انتظر حتى تظهر رسالة \"تم حفظ التعديلات بنجاح\"
10. تم إعادتك إلى صفحة الملف الشخصي
```

### التحقق:
```
✅ الصورة الجديدة تظهر في الملف الشخصي
✅ اضغط F5 (إعادة تحميل الصفحة)
✅ الصورة تظهر مباشرة (بدون تأخير)
✅ اذهب إلى صفحة أخرى ثم عد إلى الملف الشخصي
✅ الصورة موجودة ✅
```

---

## 🚀 الاختبار 2: logout ثم login بنفس الحساب

### الخطوات:
```
1. تأكد من أنك في صفحة الملف الشخصي
2. غيّر الاسم (مثلاً من \"أحمد\" إلى \"أحمد محمد\")
3. حمّل صورة جديدة (أو احتفظ بالقديمة)
4. اضغط \"حفظ التعديلات\"
5. انتظر الحفظ
6. اذهب إلى القائمة → تسجيل الخروج
7. اضغط تسجيل الخروج مرة أخرى للتأكيد
8. يجب أن تنتقل إلى صفحة Logout
9. اضغط رابط العودة أو انتظر لإعادة التوجيه إلى صفحة Login
10. ادخل بنفس الحساب مرة أخرى
11. اذهب إلى الملف الشخصي
```

### التحقق:
```
✅ الاسم الجديد يظهر (\"أحمد محمد\")
✅ الصورة الجديدة تظهر (لم تُفقد)
✅ البيانات الأخرى محفوظة (رقم الهاتف، المنطقة، إلخ)
```

## 🚀 الاختبار 3: تبديل بين مستخدمين

### الإعداد:
```
تأكد من وجود حسابين على الأقل:
- حساب عادي (Resident) - أو أي حساب عادي
- حساب Admin - أو حساب آخر
```

### الخطوات:
```
1. [مستخدم A] سجل دخول بـ الحساب الأول
   → معرّف: user@example.com
   
2. اذهب إلى الملف الشخصي
3. حمّل صورة (مثلاً صورة حمراء)
4. غيّر الاسم إلى \"أحمد\"
5. اضغط \"حفظ التعديلات\"
6. لاحظ الاسم والصورة

7. [العودة] اذهب إلى القائمة → تسجيل الخروج
8. تسجيل خروج

9. [مستخدم B] سجل دخول بـ الحساب الثاني
   → معرّف: admin@example.com
   
10. اذهب إلى الملف الشخصي
11. شاهد الاسم والصورة
    → يجب أن تكون مختلفة عن المستخدم A

12. [اختياري] حمّل صورة مختلفة (مثلاً صورة زرقاء)
13. غيّر الاسم إلى \"سارة\"
14. اضغط \"حفظ التعديلات\"

15. [العودة إلى A] اذهب إلى القائمة → تسجيل الخروج
16. تسجيل خروج

17. [مستخدم A] سجل دخول بـ الحساب الأول مجدداً
18. اذهب إلى الملف الشخصي
```

### التحقق:
```
✅ صورة A (الحمراء) موجودة
✅ اسم A (\"أحمد\") موجود
✅ صورة B (الزرقاء) لا تظهر
✅ اسم B (\"سارة\") لا يظهر
✅ كل بيانات A معزولة عن B ✅
```

## 🚀 الاختبار 4: فحص البيانات الشخصية (جهة الاتصال وتفضيلات المساعدات)

### الخطوات:
```
1. سجل دخول
2. اذهب إلى الملف الشخصي
3. اذهب إلى تعديل الملف الشخصي
4. اضغط على \"تفضيلات المساعدات\"
5. اختر عدة تفضيلات (مثلاً: غذاء، دواء، ماء)
6. اضغط زر رفع الحي (لزيادة نطاق المساعدة)
7. أدخل موقعك (مثلاً: \"الرياض\")
8. لاحظ أن الحفظ يحدث تلقائياً

9. أعد تحميل الصفحة (F5)
   → جميع التفضيلات يجب أن تظهر ✅

10. تسجيل خروج ثم دخول بنفس الحساب
    → جميع التفضيلات محفوظة ✅

11. اذهب إلى حساب مستخدم آخر
    → التفضيلات الجديدة مختلفة ✅

12. عد إلى الحساب الأول
    → التفضيلات الأصلية موجودة ✅
```

## ❌ المشاكل المحتملة والحلول:

### المشكلة 1: الصورة لا تظهر بعد إعادة التحميل
```
الحل:
1. افتح Console
2. اكتب: localStorage.clear()
3. أعد تحميل الصفحة
4. حاول تحميل الصورة مرة أخرى
5. افحص localStorage في DevTools
```

### المشكلة 2: البيانات تُفقد عند logout
```
الحل:
1. تأكد من أن resetBrowserSession تم تحديثها
2. افتح resetBrowserSession.ts
3. تحقق من وجود:
   - getCurrentUserId()
   - getAllStoredUserIds()
   - clearUserProfileData()
4. أعد تشغيل التطبيق
```

### المشكلة 3: بيانات مستخدم A تظهر عند login بـ B
```
الحل:
1. افتح DevTools → Console
2. اكتب: localStorage.clear()
3. أعد تحميل الصفحة
4. اختبر مرة أخرى

إذا استمرت المشكلة:
1. تأكد من mergeProfileAvatarOnly يدمج البيانات بشكل صحيح
2. افتح Network tab وتحقق من API response
```

## ⚠️ ملاحظات هامة:

1. **Browser Cache**: 
   - إذا رأيت بيانات قديمة، اضغط Ctrl+Shift+Delete
   - امسح جميع البيانات المحفوظة

2. **Multiple Users**:
   - تأكد من وجود حسابين على الأقل
   - اختبر الفصل بين البيانات

3. **Network Speed**:
   - إذا كان الـ API بطيء، قد تظهر البيانات المحلية أولاً
   - هذا طبيعي وجيد ✅

4. **localStorage Limits**:
   - بعض المتصفحات تحد من حجم localStorage (عادة 5-10MB)
   - الصور الكبيرة قد تسبب مشاكل
   - حجم الصورة الأقصى: 800KB
