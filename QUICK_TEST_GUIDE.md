# دليل الاختبار السريع - إصلاحات المستخدمين والـ Logout

## 🚀 الاختبار 1: تبديل من مستخدم عادي إلى Admin

### الخطوات:
```
1. افتح التطبيق وسجل دخول بـ حساب "مستخدم عادي" (resident)
   → اذهب إلى صفحة Profile
   → لاحظ الاسم المعروض (مثلاً: "محمد أحمد")
   
2. اضغط على زر "تسجيل الخروج"
   → يجب أن تنتقل إلى صفحة /logout
   → بعد 3 ثوان انتقل إلى صفحة Login
   
3. سجل دخول بـ حساب Admin
   → اذهب إلى صفحة Profile (إن أمكن) أو أي صفحة عادية
   → لاحظ الاسم المعروض (يجب أن يكون اسم Admin، ليس "محمد أحمد")
   
✅ النتيجة المتوقعة:
   - الاسم الجديد يجب أن يظهر (ليس الاسم القديم)
   - لا يجب أن ترى أي بيانات من المستخدم السابق
```

## 🚀 الاختبار 2: تسجيل الخروج من Admin

### الخطوات:
```
1. سجل دخول بـ حساب Admin
   → تأكد من أنك في صفحة admin dashboard
   
2. ابحث عن زر "تسجيل الخروج" (عادة في القائمة الجانبية)
   
3. اضغط على تسجيل الخروج
   → يجب أن تنتقل إلى صفحة /logout
   → يجب أن ترى شاشة "تم تسجيل الخروج بنجاح"
   
❌ ما لا يجب أن يحدث:
   - لا يجب أن تنتقل إلى /dashboard
   - لا يجب أن ترى بيانات المستخدم السابق
   
✅ ما يجب أن يحدث:
   - تنتقل إلى صفحة Logout
   - بعد 3 ثوان انتقل إلى /login تلقائياً
```

## 🚀 الاختبار 3: تبديل متعدد بين نفس المستخدمين

### الخطوات:
```
1. سجل دخول بـ User A
   → تحقق من بيانات User A (الاسم، الصورة، البريد الإلكتروني)
   → سجل اسم User A: _______________
   
2. تسجيل خروج
   
3. سجل دخول بـ User B (نفس النوع من الحساب أو مختلف)
   → تحقق من أن جميع البيانات مختلفة
   → سجل اسم User B: _______________
   → يجب أن تكون مختلفة تماماً عن User A
   
4. تسجيل خروج
   
5. سجل دخول بـ User A مجدداً
   → تحقق من أن البيانات هي نفس البيانات الأصلية من الخطوة 1
   → يجب أن ترى اسم User A (ليس User B)
   
✅ النتيجة المتوقعة:
   - كل مستخدم يرى بيانته الخاصة فقط
   - لا توجد مشاركة بيانات
   - البيانات محفوظة بشكل صحيح
```

#### التحقق من localStorage:
```javascript

// قبل logout
console.log(localStorage)

// بعد logout (قبل login جديد)
console.log(localStorage)

// - مفاتيح assistance_preferences_* محذوفة ✅
// - مفاتيح najat_profile_local_* محذوفة ✅
```

#### التحقق من React Query cache:
```javascript
// لا يمكن الوصول مباشرة لـ React Query cache من Console
// لكن يمكنك ملاحظة Network calls:

// 1. اذهب إلى Network tab
// 2. سجل دخول مستخدم جديد
// 3. يجب أن ترى استدعاء:
//    GET /v1/auth/me ✅

```

## 🚀 الاختبار 5: اختبار الصور والبيانات

### الخطوات:
```
1. User A يحمّل صورة شخصية
   → ملاحظة: تم حفظ الصورة
   
2. تسجيل خروج
   
3. User B يسجل دخول
   → يجب أن ترى صورة User B (ليس صورة User A)
   
4. تسجيل خروج
   
5. User A يسجل دخول
   → يجب أن ترى نفس صورة User A من الخطوة 1
   
✅ النتيجة المتوقعة:
   - كل مستخدم يرى صورته الخاصة
   - الصور لا تختلط
```

## 🔧 في حالة الفشل:

### إذا لم تنتقل إلى /logout عند تسجيل الخروج من Admin:
```
1. افتح Console في DevTools (F12)
2. ابحث عن أي رسائل error
3. تحقق من:
   - هل تم تعديل AdminShell.tsx بشكل صحيح؟
   - هل الـ route /logout موجود؟
```

### إذا كانت بيانات مستخدم قديم موجودة:
```
1. افتح DevTools
2. اذهب إلى Application → LocalStorage
3. احذف جميع مفاتيح تبدأ بـ:
   - "najat_profile_local_"
   - "assistance_preferences_"
4. أعد تحميل الصفحة وجرب مرة أخرى
```

### إذا رأيت duplicate data في UI:
```
1. افتح Console
2. اكتب: localStorage.clear()
3. أعد تحميل الصفحة
4. سجل دخول مستخدم جديد
```
