# دليل اختبار تعدد المستخدمين

## المشاكل التي تم إصلاحها:

### 1. **تفضيلات المساعدات الآن محفوظة في الـ Backend**
- **السابق**: كانت تُحفظ محلياً فقط (localStorage) ولا ترسل للـ API
- **الآن**: تُحفظ في الـ Backend عند تعديلها مباشرة

### 2. **عزل بيانات المستخدمين**
- كل مستخدم له cache منفصل في React Query (مفتاح البيانات يتضمن معرف المستخدم من JWT)
- تفضيلات المساعدات تُحمل من الـ Backend بناءً على المستخدم الحالي

### 3. **مسح البيانات عند التبديل**
- عند تسجيل الخروج والدخول بمستخدم آخر، يتم:
  - حذف الـ Token
  - مسح cache React Query
  - مسح localStorage للبيانات القديمة
  - تحميل البيانات الجديدة من الـ Backend

## خطوات الاختبار:

### الخطوة 1: اختبار تبديل المستخدمين
```
1. سجل دخول بـ المستخدم الأول
2. انتقل إلى صفحة الملف الشخصي (Profile)
3. عدّل تفضيلات المساعدات:
   ✓ فعّل خيارات مختلفة (مثل: غذاء، دواء)
   ✓ اكتب موقع محددة
   ✓ غيّر نطاق البحث
4. لاحظ "تم الحفظ" - هذا يعني حفظها في الـ Backend
5. سجل خروج
6. سجل دخول بـ المستخدم الثاني
7. انتقل إلى صفحة الملف الشخصي
   → يجب أن ترى بيانات المستخدم الثاني (تفضيلات مختلفة)
8. سجل خروج وادخل مرة أخرى بـ المستخدم الأول
   → يجب أن ترى نفس التفضيلات التي حفظتها في الخطوة 3
```

### الخطوة 2: اختبار تحديث الملف الشخصي
```
1. سجل دخول
2. اذهب إلى صفحة تعديل الملف الشخصي (Profile Edit)
3. عدّل بيانات (الاسم، رقم الهاتف، الموقع)
4. انقر حفظ
5. تحقق من:
   ✓ ظهور رسالة "تم حفظ التعديلات بنجاح"
   ✓ تحديث البيانات في الـ Backend (عبر PATCH /auth/me)
6. سجل خروج وادخل مرة أخرى
   → يجب أن ترى نفس البيانات المحدثة
```

### الخطوة 3: اختبار الصور الشخصية
```
1. سجل دخول بـ المستخدم الأول
2. عدّل الصورة الشخصية
3. لاحظ "تم الحفظ"
4. سجل خروج وادخل بـ المستخدم الثاني
   → يجب أن ترى صورة المستخدم الثاني (أو الصورة الافتراضية)
5. عدّل صورة المستخدم الثاني
6. سجل خروج وادخل مرة أخرى بـ المستخدم الأول
   → يجب أن ترى صورة المستخدم الأول
```

## ماذا إذا لم تعمل الإصلاحات؟

### تحقق من:
1. هل تم تحديث Backend API؟
   - تأكد أن endpoint `PATCH /auth/me` يقبل الحقول الجديدة:
     - `assistancePreferences` (object)
     - `assistanceLocation` (string)
     - `assistanceRadius` (number)

2. هل توجد أخطاء في Console؟
   - افتح أدوات المطور (F12)
   - اذهب إلى Network tab
   - تحقق من استدعاءات API عند الحفظ

3. هل تُحذف البيانات من localStorage؟
   - افتح Application tab
   - اذهب إلى localStorage
   - ابحث عن مفاتيح تبدأ بـ `assistance_preferences_`
   - يجب حذفها عند التبديل إلى مستخدم آخر

## معلومات اضافية:

### Query Key للـ Profile:
```typescript
['profile', 'me', '<userId>'] // مفتاح فريد لكل مستخدم
```

### API Endpoints المستخدمة:
- `GET /v1/auth/me` - لجلب بيانات المستخدم الحالي
- `PATCH /v1/auth/me` - لتحديث البيانات

### Cache Invalidation:
- عند تسجيل الخروج: تُحذف جميع مفاتيح البيانات من React Query
- عند التبديل إلى مستخدم آخر: تُحمل البيانات الجديدة تلقائياً
